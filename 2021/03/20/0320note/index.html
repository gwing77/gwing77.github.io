ES6模块化

模块化介绍和语法

模块化就是把一个大的程序文件拆分成许多小的文件，之后组合起来

  1. 防止命名冲突
  2. 代码复用
  3. 高维护性
    ES6之前的模块化规范有
  4. CommonJS => NodeJS/Browserify
  5. AMD => requireJS
  6. CMD =》 seaJS

ES6的模块化功能主要有两个命令构成（我一开始学elementjs想要看的应该就是这个功能，这视频真的是卒章显志啊）：

  1. export命令用于规定模块的对外接口
  2. import命令用于输入其他模块提供的功能
    ES6模块化的使用
  3. 写代码的时候一般把import语句都写在一个app.js文件里面，在我们的html中script标签（type=’module’）直接引入app.js文件使用
  4. 但是实际使用过程中，很多浏览器可能对ES6支持的不是很友好，ES6对npm安装的一些模块不能直接导入，因此要使用babel
  1. 安装工具babel-cli(babel的命令行工具) babel-preset-env（预设包，将ES6转化为ES5语法） browserify（打包工具browserify很简单，实际项目会使用webpack需要很多的配置）
  2. 代码转换 局部安装npx，全局安装babel 源文件目录 -d 存到那个目录 预设参数（可以命令行传参或者写babel的默认配置文件）
  3. 打包 打包之后的文件引入html才是可以使用的
  4. 注意每次修改了源文件要重新打包，因为我们最后引入的是bundle.js
使用npm包(举例jquery）
  1. npm i jquery
  2. 在app.js中导入jquery
